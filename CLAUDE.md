## Memory

Read `memory/MEMORY.md` at session start for persistent context across conversations.

When you learn something worth keeping — a lesson, a project detail, a preference — save it to the appropriate memory file and update the index. Use `/dmg` at the end of sessions to sync everything.

## Memory types

- **project** — current state, file paths, open threads for a specific project
- **feedback** — lessons learned: what to do/avoid and why; format as rule + **Why:** + **How to apply:**
- **user** — facts about the user's role, preferences, expertise level
- **reference** — pointers to external resources (dashboards, docs, tickets)
